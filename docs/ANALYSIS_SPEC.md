# 解析・評価仕様

## 目的

解析結果の意味と変更時の確認事項を、実装コードとは別に追跡できるようにする。この文書は現行コードの要約であり、数値の最終的な正本は記載したソースファイルと自動テストである。

## 解析単位とバージョン

- 履歴へ保存する解析基準のバージョンは `src/types/analysisHistory.ts` の `ANALYSIS_VERSION` で管理する。
- 評価式、閾値、測定フレームの定義を変更した場合は、バージョンを更新する。
- 異なる解析バージョンの結果を比較する場合は、同じ基準ではない可能性を明示する。

## 動画と座標

- ユーザーが選んだ動画は `URL.createObjectURL` でブラウザ内から読み込む。
- 現行実装は動画本体をFirebase Storageなどへアップロードしない。
- MediaPipeの骨格推定結果を動画のピクセル座標へ変換して解析する。
- 撮影角度、遮蔽、画角、明るさ、解像度、fpsが精度へ影響する。

## 手動計測

### cm/px

画面上の既知距離2点間のピクセル距離を `px`、実距離を `knownCm` とする。

```text
cmPerPx = knownCm / px
```

### 最高到達点

リング位置と指先位置の縦方向差を使う。

```text
最高到達点 = リング高 + (リングY - 指先Y) × cmPerPx
```

### 滞空時間と推定ジャンプ高

離地と着地として保存した2時刻の絶対差を滞空時間とする。

```text
推定ジャンプ高[m] = 9.81 × 滞空時間² / 8
```

### 球速

ボールの2点間距離をcm/pxで実距離へ変換し、2時刻の差で割ってkm/hへ変換する。

## 自動フォーム解析

### 踏切時間

現行コードでは、`jumpPhaseEngine`が検出した沈み込み開始から離地までを近似的な踏切時間としている。以前の会話で決めた「最後の踏み込みで両足が接地した瞬間から、両足が離地した瞬間まで」と完全に一致するかは、検証動画で追加確認が必要である。

表示用の星評価は現行コードで次の境界を使う。

| 踏切時間 | 星 |
|---:|---:|
| 0.35秒以下 | 5 |
| 0.45秒以下 | 4 |
| 0.60秒以下 | 3 |
| 0.80秒以下 | 2 |
| 0.80秒超 | 1 |

### 空中姿勢

最高点の前後0.12秒を対象に、次の3要素を合成する。

| 要素 | 重み | 現行基準 |
|---|---:|---|
| 打つ側の肩から反対側の足先までの一直線性 | 45% | 基準線から腰・膝までの距離を全身ライン長で正規化。小さいほど高評価 |
| 全身ラインの角度 | 35% | 画面水平を暫定的なネット水平とし、45°を理想、40〜50°を高評価 |
| 余計な動き | 20% | 体幹と足の短時間の方向反転を数え、少ないほど高評価 |

ネットの実線を検出する機能は現時点ではないため、「ネット基準」は画面水平による代替である。カメラが傾いている動画では誤差が生じる。

空中姿勢の0〜100点は次の境界で星へ変換する。

| スコア | 星 |
|---:|---:|
| 90以上 | 5 |
| 75以上 | 4 |
| 60以上 | 3 |
| 40以上 | 2 |
| 40未満 | 1 |

### 肩の傾き

- 打つ側の肩が6°以上高い: 星5
- 打つ側の肩が0°以上6°未満: 星4
- 逆方向4°以内: 星3
- 逆方向8°以内: 星2
- それ以上の逆傾斜: 星1

### 着地

着地は解析・カテゴリ評価・改善コメント・履歴保存の対象に含めるが、スパイク総合スコアには含めない。`formSummary`と`analysisConfidence`の両方で、総合スコアは助走・踏切・空中姿勢・スイングの測定可能なカテゴリだけから算出する。

この集計方法への変更は解析バージョン`1.1.0`から適用する。`1.0.0`以前の履歴は着地評価が総合スコアへ含まれている可能性があるため、同一基準として比較しない。

## 評価基準を変更する際の必須作業

1. 変更前後の式と理由をIssueまたはPRへ記載する。
2. 境界値の単体テストを更新する。
3. 正解データ付き検証動画で回帰確認する。
4. `ANALYSIS_VERSION`を更新する。
5. 過去バージョンとの比較時の扱いを確認する。
