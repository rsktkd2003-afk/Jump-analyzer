# 運用・障害対応

## 障害の切り分け

1. 公開URLと発生時刻を記録する。
2. 端末、OS、ブラウザ、PWAか通常タブかを記録する。
3. 動画のfps、解像度、撮影方向を匿名情報として記録する。
4. ブラウザの開発者ツールでコンソールエラーを確認する。
5. 同じ動画で再現するか確認する。
6. シークレットウィンドウでPWAキャッシュの影響を切り分ける。
7. Firebase Authentication、Firestore、Hostingの状態を確認する。

動画、骨格座標、UID、表示名、履歴メモはIssueへ貼らない。

## 重大度

| 重大度 | 例 | 初動 |
|---|---|---|
| 高 | 他人の履歴を閲覧できる、秘密情報流出 | 公開停止を検討し、認証・Rulesを最優先で確認 |
| 中 | 解析不能、誤った結果を広範囲に表示 | 影響版を特定し、直前版へのロールバックを検討 |
| 低 | 特定端末の表示崩れ、軽微な文言 | Issue化し通常の修正フローで対応 |

## PWAキャッシュ問題

1. 更新通知が表示されるか確認する。
2. 「更新する」を押して再読み込みする。
3. 改善しない場合は、ブラウザのサイトデータとService Workerを削除して再確認する。
4. `dist`へ古い生成物が混在していないか、クリーンビルドで確認する。

## ロールバック

Firebase ConsoleのHostingリリース履歴から、直前に正常だったバージョンを再リリースする。ロールバック後も原因修正は別ブランチとPRで行い、問題のあるコミットを履歴から破壊的に削除しない。

## バックアップ

- ソースコードと仕様書はGitHubで履歴管理する。
- Firestoreの本番バックアップ方式と保存期間は未設定である。利用者が増える前に、Firebase/Google Cloud側で定期エクスポート方針を決める。
- 検証動画は公開GitHubへ置かず、同意とアクセス制限を確認できる保存先で管理する。
