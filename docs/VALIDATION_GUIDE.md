# 精度検証ガイド

## 目的

「同じ動画で同じ結果が出るか」と「人が確認した正解値へどの程度近いか」を継続的に測る。

## 検証データの条件

最低限、次の条件を組み合わせて用意する。

- 30fps / 60fps
- 正面 / 側面 / 斜め
- 明るい / 暗い
- 全身が常に入る / 一部遮蔽あり
- 1人 / 背景に別の人物がいる
- スパイクジャンプの成功例 / 崩れた例

人物が特定できる動画は公開リポジトリへコミットしない。本人の同意を得て、アクセス制限された場所で管理する。

## 正解値の作成

1. 2人以上で離地・最高点・着地フレームを確認する。
2. 判断が割れた場合は両方の候補を残し、確定理由を記録する。
3. 既知距離または実測した最高到達点がある場合だけ、cm精度の検証へ使う。
4. フォーム評価は、評価者と根拠を記録する。

## 記録方法

`docs/validation/validation-results.csv`をコピーして使用する。動画そのものではなく、外部管理用の匿名IDだけを記録する。

主な誤差:

```text
絶対誤差 = |自動測定値 - 正解値|
平均絶対誤差 = 絶対誤差の合計 / 件数
フレーム誤差 = |自動検出フレーム - 正解フレーム|
```

## 合格基準

現時点で、最高到達点やフェーズ検出の正式な許容誤差は確定していない。根拠なく数値を決めず、最初の検証データを集めた後に用途別の合格基準を決める。

ただし、コード変更時は最低限次を満たすこと。

- 単体テストがすべて成功する。
- 基準動画で、意図していない評価変化がない。
- 測定不能な動画を無理に数値化しない。
- 変更による差がある場合、その理由をPRへ記録する。

## 不具合動画の記録

- 匿名動画ID
- 発生日とアプリバージョン
- 端末・ブラウザ
- fps・解像度・撮影方向
- 期待結果と実際の結果
- コンソールエラー
- 再現率
- 原因と修正コミット
