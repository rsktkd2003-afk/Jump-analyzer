# セキュリティ方針

## 対象

Jump Analyzerの認証、Firestore、動画処理、依存パッケージ、PWA配信を対象とする。

## 脆弱性の連絡

公開Issueへ認証情報、個人情報、動画、攻撃手順の詳細を投稿しないでください。リポジトリ所有者へ非公開の連絡手段で報告してください。現在、専用のセキュリティ連絡先は未設定です。

## 現在の保護

- Firestoreの履歴は `users/{uid}/analysisHistories/{analysisId}` に保存する。
- Firestore Rulesは、ログイン中の本人だけが自分のユーザードキュメントと履歴を操作できるよう制限する。
- 履歴作成時は `userId`とドキュメントIDの一致を検証する。
- `.env*`はGit管理対象外とし、`.env.example`だけを共有する。
- 動画はブラウザ内のObject URLから解析し、現行コードでは外部ストレージへアップロードしない。
- アプリの障害記録には、動画、骨格座標、UID、表示名、メモを含めない。

## 開発者向け確認

```powershell
npm ci
npm run check
npm audit
```

Firestore Rulesを変更した場合は、Firebase Emulatorで次を確認する。

- 未ログイン状態では読み書きできない。
- ユーザーAがユーザーBの履歴を読めない、作れない、更新できない、削除できない。
- 履歴の `userId`を別UIDへ書き換えられない。
- 履歴のドキュメントIDと `analysisId`が異なる状態で作成できない。

## 未対応事項

- Firestore Rulesの自動エミュレータテスト
- 永続的なエラー監視サービス
- 専用のセキュリティ連絡先
- アカウントと全履歴を一括削除する正式な導線の確認
